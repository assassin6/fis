$(function(){
    //加载需要用到的接口模块，多个模块用“,”隔开，并且在对应位置声明一个对应模块的变量(可与模块名相同),如下示例引用的ms.people,ms.mbbs和test三个模块，对应的变量声明为mpeople,mbbs和test。test模块为模板中引用的一个js文件，如下声明中使用的标签可看出该模块位置在模板js文件夹下，注意ms标签只能在模板文件html文件下解析，如果在js文件中使用模板中的js模块时，可将该模块在公共头部文件中定义一个全局变量在js中使用，另外ms为基础层模块，使用ms.code加载验证码需要加载ms模块，声明变量为ms
  ms.load(["ms.people","ms.mbbs","{ms:globalskin.url/}/js/test"],function(mpeople,mbbs,test) {
    //需要用的接口方法
    mpeople.register($("form").serialize(),function(returnJson){
      alert(JSON.stringify(returnJson));
    })
  })
})

ms.people: API

register

用户注册

用户可以用名称、手机号、邮箱三者之一进行注册
几种注册流程的形式：
1、普通用户名称、登录密码，优先用户名注册,登录密码最长度范围6～12个字符；
2、邮箱、邮箱验证码、登录密码（邮箱必须是可接收验证码）；
3、手机号、短信验证码、登录密码；
注意： 1、注册页面必须存在图片验证码
2、如果需要接收验证码操作，需要使用“发送验证码”配合使用才能完成注册流程

调用方法

register(data,function(returnJson){...});

参数说明

名称 类型 是否必须 描述
peoplePhone string 可选

手机号
peopleName string 可选

用名称 用户名长度在3～12个字符之间，只能是字母数字混合
peopleMail string 可选

邮箱
peoplePassword string 必填

注册密码
rand_code strings 必填

验证码
回调方法 function 必填

返回值（returnJson）

返回值说明

名称 描述
resultMsg

提示信息
code

错误编码
result

true成功、false失败

请求实例

...
<form name="registe">
  <input type="text" name="peopleName" /> <!--注册用户名-->
  <input type="password" name="peoplePassword"/> <!--注册密码-->
  <input type="password" name="rePassword"/> <!--确认注册密码，需要做前端验证，此接口不提供验证-->
  <input type="text"  name="rand_code" /> <!--验证码-->
  <img id="registeCode"/> 
</form>
